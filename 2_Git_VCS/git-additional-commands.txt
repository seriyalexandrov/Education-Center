1)  Посмотреть историю того, как пользователь перемещался по коммитам. 
	Помогает найти "потерянный" коммит.
	
	git reflog
	
2) git reset. Читаем сами: https://habrahabr.ru/post/203282/

	Чаще всего используется для того, чтобы откатить изменения в репозитории на нужный коммит.
	Тогда выполняется такая команда:
	
	git reset --hard <commit id>
	
3) В то место, где мы сейчас находимся можно применить ЛЮБОЙ существующий коммит. 

	git cherry-pick <commit id>
	
4) Незакоммиченные изменения сделанные локально можно "прятать". Например для того чтобы переключиться на другую ветку, иначе git не даст это сделать. Очень просто:

	git stash
	
	Дальше смотрим git status, видим, что все изменения пропали.
	Чтобы применить спратанные изменения, достаточно написать
	
	git stash apply
	
	Изменения будут применены даже если это совершенно другое место.
	Бывает, что спрятанных изменений накопилось много, и надо выбрать нужное. Для этого смотрим их:
	
	git stash list
	
	Выбираем нужный и применяем:
	
	git stash apply stash@{<номер>}
